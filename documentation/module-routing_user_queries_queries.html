Module: routing/user/queries/queries

The module contains a set of provided short hand mongoose query calls. The queries provided should be used by the user side of the front-end.

Methods

(inner) getOneElement(database, tag, selectTerm)

The query looks for the element with the given tag. This should be used for all communication with the server. However, if there is a need to query the database by the _id, see getOneElementById
Parameters:
Name Type Attributes Description
database mongoose.Schema the database as defined in a mongoose schema
tag json the tag is the search term used for the query, an arbitrary value used to start the whole chatbot.
selectTerm string the fields in the database to be shown in the requested JSON document. If the field is not specified here, it will not be shown in the document. An empty string shows all fields.
callback.e string <nullable>
 an error message, null if there is no error
callback.element json the database element returned by the query.
