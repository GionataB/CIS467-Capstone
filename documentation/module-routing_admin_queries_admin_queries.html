Module: routing/admin/queries/admin_queries

The module contains a set of provided short hand mongoose query calls. The queries provided should be used by the admin side of the front-end.

Methods

(inner) deleteAllElements(database)

The query is shorthand for deleting all elements in the database. No search term has to be passed, as it defaults to any element (the empty set '{}').
The callback from the query returns, in this case, the whole collection. At the end of this call, the collection is effectively wiped out, so be careful in calling it.
Parameters:
Name Type Attributes Description
database mongoose.Schema the database as defined in a mongoose schema.
callback.e string <nullable>
 an error message, null if there is no error.
callback.elements json the deleted collection of elements.

(inner) deleteField(database, searchTerm, field)

The query deletes the requested fields, or sub-documents, in a document. The field is a list of values: it is possible to remove multiple fields in one call.
However, it is not possible to remove all the occurences of field from different documents in the collection. For that, see deleteFieldFromAll.

 an error message, null if there is no error.

(inner) getAllElements(database, searchTerm, selectTerm)

The query gets all elements in the database. If an error occurs, e will be set to true.
Generally, an error means that the server had a problem. It does not return an error in the case the database is empty, as 'empty' is a valid response to querying all elements.

(inner) postManyElements(database, newElementArr)

The query takes an array of documents to save in the database. The query runs the mongoose schema validators for each entry, and saves to the database all entries that pass the validators.
The storing of the documents is done in bulk, with only one operation. If ordered = false, then the error will reports all the documents that have failed the validators. If true, stop the operation at the first document that fails the validators, not saving anything on the database.
Parameters:
Name Type Attributes Description

newElementArr Array.<json> an array of populated Schema objects, each containing a document to save to the database.
callback.err bool <nullable>
 an error message, false if there is no error.

(inner) updateOneAnswer(database, searchTerm, field, fieldID, index, update)

The query is used to update a nested sub-document. Due its complexity, it requires a lot of information to find the correct field to update.
Note: first, the entire document is pulled from the database, then the sub-document with the field is selected, the field updated, and then the new document is saved back.
Because of that the operation is not atomic, so beware of changes in the document happening between pulling it from the database, and saving the new one.
Parameters:
Name Type Attributes Description
database mongoose.Schema the database as defined in a mongoose schema.
searchTerm json JSON containing the terms used for the database search.
field String the name of the sub-document, or the array that contains the field to update.
fieldID mongoose.ObjectID the _id associated with the desired field.
index Number | String the position of the field in the array of answers.
update JSON a field:value pair that specifies the field to update with its new value.
callback.e string <nullable>
 an error message, null if there is no error
callback.element json the updated database element.

(inner) updateOneDocument(database, searchTerm, update)

The query updates a field at the top level of the document. If an error occurs, e will be set to true.
Generally, that means that the element was not found, or the update does not pass the database's constraints.
This does not update a sub-document. If you want to modify a sub-document, like a field in one index in answers, see updateOneAnswer.
The operation is atomic and operates on the database without pulling the original element.
Parameters:
Name Type Attributes Description
database mongoose.Schema the database as defined in a mongoose schema.
searchTerm json JSON containing the terms used for the database search.
update json JSON containing the field(s) to update/add.
callback.e string <nullable>
 an error message, null if there is no error.
callback.element json the updated database element.
